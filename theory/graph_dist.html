Алгоритм Флойда.

Самый простой алгоритм на нахождение кратчайшего пути. Работает за куб, но при этом находит кратчайшие расстояния между каждой парой вершин, в отличии от других алгоритмов.

Осторожно при реализации, лично я всегда путаю и ставлю третий цикл не туда.

Алгоиртм Дейкстры.

Самый главный и популярный алгоритм в данной теме. Есть две вариации за n^2 и nlogn. Для обучения стоит написать вначале первую, а уже потом перейти на вторую.

Второй вариант также можно писать несколькими способами. Через сет или приоритетную очередь. Вроде через очередь лучше, но я ее не знаю, и всегда пишу через сет. На всякий случай прилагаю свой код:

// Массив v - список смежности в виде пары {вершина,стоимость}
// Массив дист изначально должен быть заполнен +бесконечностями.

void dijkstra(int s,vector<int> &dist) {
	dist[s]=0;
	set<pair<int,int> > q;
	q.insert({0,s});
	while(!q.empty()) {
		int cur=q.begin()->second;
		q.erase(q.begin());
		for(int i=0;i<v[cur].size();i++) {
			int to = v[cur][i].first, len = v[cur][i].second;
			if (dist[cur]+len < dist[to]) {
				q.erase({dist[to],to});
				dist[to] = dist[cur]+len;
				q.insert({dist[to],to});
			}
		}
	}
}

Алгоритм Форда-Белламана.

Данный алгоритм нужен в случае наличия отрицательных весов ребер в графе.

Shortest Path Faster Algorithm. SPFA.

Малоизвестный алгоритм. Насколько я понимаю, хорошо работает в случайных графах, но возможно придумать контртесты.

Можно изучить про него для общего развития.
